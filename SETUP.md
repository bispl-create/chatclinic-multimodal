# ChatClinic — setup guide

End-to-end bootstrap for anyone cloning this repo (professor, classmates, CI).
Targets Linux with at least one NVIDIA GPU (tested on RTX 3090, CUDA 12.2 driver).

## TL;DR

```bash
git clone <THIS_REPO_URL> chatclinic-class
cd chatclinic-class
bash bootstrap.sh              # installs deps, clones backends, downloads weights
bash start_demo.sh             # launches backend + frontend + pre-warms GPU
# open http://127.0.0.1:3000
```

If you don't want the helper scripts, follow the manual steps below — they're
the same thing, just explicit.

---

## 0. What's in the repo (versus what's fetched during setup)

This repo is intentionally **code-only**. Three categories of assets are
downloaded on first setup (total ~15 GB) and ignored by git:

| Path | Fetched from | Size | Why it's not in git |
|---|---|---|---|
| `external_backends/` | GitHub (4 upstream repos) | ~1 GB | License hygiene; get the canonical sources |
| `external_weights/` | Google Drive + Hugging Face | ~6 GB | Binary model weights |
| `demo_images/` | Generated by a script from pydicom/nibabel test data | ~2 MB | Trivially reproducible |
| `outputs/` | Produced by the test harness | ~3 MB | Trivially reproducible |
| `chatclinic-multimodal/node_modules/` | npm | ~425 MB | Standard JS convention |
| `chatclinic-multimodal/.next/` | Next.js build cache | varies | Regenerated |

Place `OPENAI_API_KEY=sk-…` into `chatclinic-multimodal/.env` (the file is gitignored).

## 1. System prerequisites

```bash
# Linux with NVIDIA driver ≥ 525 (supports CUDA 12.x). Confirm:
nvidia-smi

# Tooling:
sudo apt install -y git curl build-essential   # (or dnf/pacman)

# Install uv (fast Python package manager) — one line, no sudo:
curl -LsSf https://astral.sh/uv/install.sh | sh
export PATH="$HOME/.local/bin:$PATH"

# Install Node.js 20+ (for the Next.js frontend). Either your distro's node
# or nvm:
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.40.1/install.sh | bash
source ~/.nvm/nvm.sh
nvm install 22
nvm use 22

# Install gdown (Google Drive downloader) for the CoreDiff checkpoint:
uv tool install gdown
# OR: pip install --user gdown
```

---

## 2. Clone the 4 external backend repos

These are the unmodified upstream projects. We load their models in-process
but keep their code under `external_backends/` for license clarity.

```bash
cd external_backends
git clone https://github.com/qgao21/CoreDiff.git                CoreDiff
git clone https://github.com/mirthAI/Fast-DDPM.git              Fast-DDPM
git clone https://github.com/microsoft/SNRAware.git             SNRAware
git clone https://github.com/ileri-oluwa-kiiye/SharpXR.git      SharpXR
cd ..
```

Only **SNRAware** needs its own isolated venv installed (its deps clash with
the ChatClinic runtime). We do this once:

```bash
cd external_backends/SNRAware
uv sync           # installs torch 2.9 + cu128, mkl, pywavelets, etc.
cd ../..
```

---

## 3. Download pretrained weights

```bash
mkdir -p external_weights/{CoreDiff/checkpoint,FastDDPM,SNRAware}

# 3a) CoreDiff — official EMA checkpoint on Google Drive
#     Paper: Gao et al., IEEE TMI 2024 — Mayo 2016, 25 % dose
gdown --folder "https://drive.google.com/drive/folders/1rGb34H_6ktP79vMYYJOLSoCE3579TDZ5" \
      -O external_weights/CoreDiff

# 3b) Fast-DDPM — Hugging Face release
#     Paper: Jiang et al., JBHI 2025
uv tool install huggingface_hub            # or: pip install --user huggingface_hub
huggingface-cli download SebastianJiang/FastDDPM \
    ckpt_LDFDCT.pth ckpt_PMUB.pth ckpt_BRATS.pth \
    --local-dir external_weights/FastDDPM

# 3c) SNRAware — Microsoft, 3 model sizes
huggingface-cli download microsoft/SNRAware \
    --local-dir external_weights/SNRAware

# 3d) TorchXRayVision ResNetAE — auto-downloads to ~/.torchxrayvision/
#     on first call. No manual step needed.
```

Verify:
```bash
ls external_weights/CoreDiff/checkpoint/ema_model-150000
ls external_weights/FastDDPM/ckpt_*.pth
ls external_weights/SNRAware/{small,medium,large}/*.pts
```

All 5 files should exist before step 6.

## 4. Install the ChatClinic backend into the SNRAware venv

We reuse SNRAware's venv for the whole app so everything runs on one Python +
one CUDA build:

```bash
uv pip install --python external_backends/SNRAware/.venv \
    'fastapi>=0.115,<1.0' 'uvicorn>=0.30' 'python-multipart' \
    'pydantic>=2.8,<3.0' 'pysam>=0.23' 'openpyxl' 'Pillow>=10.0' \
    'nibabel>=5.2' pydicom requests torchxrayvision
```

## 5. Install the frontend

```bash
cd chatclinic-multimodal
npm install --workspace webapp
cd ..
```

## 6. Configure your OpenAI API key

```bash
cp chatclinic-multimodal/sample.env chatclinic-multimodal/.env
# then edit chatclinic-multimodal/.env and set:
# OPENAI_API_KEY=sk-...
```

The `.env` file is gitignored — it will never be committed.

## 7. Generate demo inputs (~5 seconds)

Creates `demo_images/clean/` (real CT + MR DICOMs + NIfTI) and
`demo_images/noisy/` (19 degraded versions with realistic noise models):

```bash
cd chatclinic-multimodal
uv run --project ../external_backends/SNRAware python3 \
    -m plugins.medical_restoration_common.build_demo_inputs
cd ..
```

---

## 8. Launch the demo

Two terminals.

**Terminal 1 — backend (pinned to GPU 0):**
```bash
cd chatclinic-multimodal
CUDA_VISIBLE_DEVICES=0 uv run --project ../external_backends/SNRAware \
    python3 -m uvicorn app.main:app --host 127.0.0.1 --port 8001
```

**Terminal 2 — frontend + pre-warm GPU:**
```bash
cd chatclinic-multimodal
npm --workspace webapp run dev &
# wait a few seconds for Next.js to compile
until curl -s http://127.0.0.1:3000 > /dev/null; do sleep 2; done

# Load all 4 in-process models onto the GPU so nvidia-smi shows ~1.8 GB:
uv run --project ../external_backends/SNRAware python3 \
    -m plugins.medical_restoration_common.prewarm \
    --backend http://127.0.0.1:8001 \
    --demo-dir ../demo_images
```

Open http://127.0.0.1:3000. Follow the scenarios in
[DEMO_SCENARIOS.md](DEMO_SCENARIOS.md).

---

## 9. (Optional) Rebuild the scenario outputs

18 pre-rendered before/after scenarios live under `outputs/` (also gitignored).
Regenerate with:

```bash
cd chatclinic-multimodal
uv run --project ../external_backends/SNRAware python3 \
    -m plugins.medical_restoration_common.run_all_scenarios
# takes ~90 s; writes outputs/{s01..s18}/ with before.png, after.png, compare.png,
# metrics.json, summary.md, plus outputs/INDEX.md
```

## Troubleshooting

| Symptom | Fix |
|---|---|
| `torch.cuda.is_available() == False` | Use the SNRAware venv — its torch 2.9+cu128 matches the driver. A `conda` env with a mismatched torch won't see the GPU. |
| `No module named 'app'` when starting uvicorn | Run from `chatclinic-multimodal/` (where `app/` lives), not from repo root. |
| `gdown` blocks the Google Drive download | Run `gdown --folder <URL> --no-check-certificate` or download manually and place the `ema_model-150000` file under `external_weights/CoreDiff/checkpoint/`. |
| `huggingface-cli download` prompts for login | The repos are public; ignore the login prompt or run `huggingface-cli login` with a free token. |
| Port 3000/8001 in use | `fuser -k 3000/tcp 8001/tcp` |
| `@xray_denoise` falls back to Fast-DDPM | Reinstall torchxrayvision into the venv, then restart uvicorn: `uv pip install --python external_backends/SNRAware/.venv torchxrayvision` |
| Webapp build fails with workspace errors | Use `npm --workspace webapp run dev` (not `cd webapp && npm run dev`). The repo uses npm workspaces. |
| `nvidia-smi` shows only ~500 MB on GPU 0 | You forgot step 8's pre-warm. Models load lazily on first @command. |

## What's running where after setup

| Process | Port | VRAM | Holds |
|---|---|---|---|
| uvicorn (FastAPI) | 8001 | ~1.8 GB on GPU 0 | CoreDiff + Fast-DDPM (3 ckpts) + ResNetAE |
| Next.js dev server | 3000 | 0 | UI |
| SNRAware subprocess | transient | ~1.3 GB per call | MRI denoising (spawned per `@mri_denoise`) |

Single RTX 3090 (24 GB) covers everything with ~22 GB headroom.
